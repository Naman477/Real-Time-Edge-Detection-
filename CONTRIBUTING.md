# Contributing to Real-Time Edge Detection

Thank you for your interest in contributing to the Real-Time Edge Detection project! We welcome contributions from the community to help improve this application.

## Getting Started

1. Fork the repository
2. Clone your forked repository
3. Create a new branch for your feature or bug fix
4. Make your changes
5. Test your changes thoroughly
6. Commit your changes with a descriptive commit message
7. Push your changes to your fork
8. Submit a pull request

## Development Setup

### Prerequisites

- Android Studio with NDK support
- Android SDK API level 24 or higher
- Git

### Building the Project

1. Open the project in Android Studio
2. Wait for Gradle to sync and download dependencies
3. Build the project using `Build > Make Project`

## Code Style Guidelines

### Kotlin
- Follow the official [Kotlin Coding Conventions](https://kotlinlang.org/docs/coding-conventions.html)
- Use meaningful variable and function names
- Keep functions small and focused
- Use extension functions appropriately

### C++
- Follow the [Google C++ Style Guide](https://google.github.io/styleguide/cppguide.html)
- Use meaningful variable and function names
- Initialize variables upon declaration
- Use const and constexpr where appropriate

### General Principles
- Write clean, readable, and maintainable code
- Add comments for complex logic
- Keep pull requests focused on a single feature or bug fix
- Write unit tests for new functionality when applicable

## Reporting Issues

If you encounter any bugs or have feature requests, please create an issue on GitHub with the following information:

1. A clear and descriptive title
2. Steps to reproduce the issue
3. Expected behavior
4. Actual behavior
5. Screenshots if applicable
6. Device and Android version information

## Pull Request Guidelines

1. Ensure your code follows the style guidelines
2. Include tests for new functionality
3. Update documentation as needed
4. Write a clear and descriptive pull request description
5. Reference any related issues
6. Keep pull requests focused and atomic

## Areas for Improvement

We're particularly interested in contributions for:

- Performance optimizations
- Additional edge detection algorithms
- UI/UX improvements
- Bug fixes
- Documentation enhancements
- Testing improvements

## Questions?

If you have any questions about contributing, feel free to create an issue asking for clarification or guidance.
